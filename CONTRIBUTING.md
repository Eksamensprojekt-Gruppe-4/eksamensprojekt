# Contributing - intern

## Workflow

Vi arbejder med **feature branches** og **pull requests**

### 1. Opret branch

Branch navngives beskrivende

```bash
git checkout -b feature/beskrivende-navn
```

### 2. Udvikling

Test din kode lokalt inden commit
Sørg for at projektet kan buildes og kør tests

### 3. Commit

Brug en klar og beskrivende commit besked

```bash
git commit -m "Tilføjer oprettelse af projekter med validering"
```

### 4. Push

```bash
git push origin feature/navn
```


### 5. Pull Request

Opret en Pull Request til `main` og tilføj en reviewer

## Vigtigt

Hold den gode tone og sikre god kode kvalitet
